# Code of Conduct

## Our Pledge

We pledge to make participation in the Dot community a respectful, professional, and harassment-free experience for everyone, regardless of background, identity, or experience level.

Dot is built on simplicity and intentionality.  
Our community should reflect the same qualities.

---

## Our Standards

**Positive behavior includes:**

- Communicating with clarity and respect
- Giving constructive, specific feedback
- Accepting responsibility for mistakes without defensiveness
- Prioritizing the long-term health of the project over personal preferences
- Maintaining a professional tone in all public interactions

**Unacceptable behavior includes:**

- Harassment, discrimination, or personal attacks of any kind
- Sexualized language or imagery
- Sustained disruption of discussions or pull requests
- Publishing private information without consent
- Deliberate dismissiveness or condescension toward contributors
- Conduct inappropriate for an open professional environment

---

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub Issues and Discussions
- Pull Requests
- Code review comments
- Any public interaction representing Dot or its author

---

## Enforcement Responsibilities

The project maintainer is responsible for clarifying and enforcing these standards.

Contributions, comments, or interactions that violate this Code of Conduct may be removed, edited, or rejected without obligation to explain.

Enforcement decisions will be proportionate, documented where appropriate, and final.

---

## Enforcement Guidelines

Responses will follow a graduated approach:

1. **Correction** — Private message clarifying the issue and expected behavior
2. **Warning** — Formal notice with documented expectations going forward
3. **Temporary Restriction** — Limited interaction for a defined period
4. **Permanent Removal** — Removal from all project spaces for repeated or severe violations
