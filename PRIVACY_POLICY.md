# Privacy Policy

**Effective Date:** March 2026  
**Last Updated:** March 2026

Dot is a minimal day-marking calendar for Android.  
This Privacy Policy describes how information is handled when you use the app.

The short version: Dot collects nothing. Dot sends nothing. Your data never leaves your device.

---

## 1. Data Collection

Dot does **not** collect, transmit, store remotely, or sell any personal data.

The app does not collect:

- Personal identifiers of any kind
- Location data
- Usage analytics or telemetry
- Advertising or device identifiers
- Biometric data
- Keystrokes or screen content
- Any information about which days you mark

All data — specifically, the set of dates you have marked — is stored locally on your device using Android DataStore. It exists only in your app's private storage sandbox.

---

## 2. Local Storage

The only data Dot stores is a set of date strings representing marked days, in the format `YYYY-MM-DD`.

This data:

- Lives in your device's private app storage
- Is never transmitted to any server
- Is never shared with any third party
- Is permanently deleted when you uninstall the app

Dot has no backend. There are no servers. There is no account system.

---

## 3. Permissions

Dot requests no permissions.

It does not request access to:

- Camera
- Microphone
- Contacts
- Location
- Storage
- Network
- Notifications

The app widget functions entirely within the standard Android widget framework and requires no additional permissions.

---

## 4. Third-Party Services

Dot uses no third-party SDKs.

It does not include:

- Analytics frameworks (Firebase, Mixpanel, etc.)
- Advertising SDKs
- Crash reporting tools
- Cloud database services
- Social login providers

The app operates entirely offline.

---

## 5. Children's Privacy

Dot does not knowingly collect any data from any user, including children under the age of 13.  
Since no personal data is collected at all, no special processing applies.

---

## 6. App Widget

The Dot home screen widget reads from the same local DataStore as the main app. No data is shared externally as part of widget functionality. Tapping a day in the widget writes to local storage only, identically to tapping in the app.

---

Any changes will be reflected in the "Last Updated" date at the top of this document.  
Continued use of the app after changes constitutes acceptance of the updated policy.
